# 测试计划

## 测试目标

测试目标是验证系统从视频接入、AI 检测、后端事件生成、告警处置、日志追溯到统计展示的核心闭环。当前阶段以框架可启动、接口边界可访问和文档一致性为主，业务测试用例随功能实现逐步落地。

## 测试范围

| 测试类型 | 覆盖内容 |
| --- | --- |
| 功能测试 | 登录、摄像头、员工、区域、告警、日志、考勤 |
| 接口测试 | REST API、统一响应格式、错误码、分页 |
| 视频流测试 | 摄像头接入、播放地址、AI 拉流 |
| AI 上报测试 | 检测结果格式、后端接收、事件生成 |
| 告警测试 | 陌生人、区域入侵、未戴头盔、摔倒、跑动 |
| 权限测试 | 未登录、普通用户、管理员权限 |
| Jenkins 构建测试 | 后端检查、前端构建、AI 编译检查 |

## 接口测试要求

1. 成功响应包含 `code`、`message`、`data`、`requestId`。
2. 参数错误返回 `400`。
3. 未登录返回 `401`。
4. 无权限返回 `403`。
5. 资源不存在返回 `404`。
6. 业务校验失败返回 `422`。

## 视频流测试要求

1. 摄像头原始流可被视频流服务接入。
2. 前端播放地址可正常播放。
3. AI 服务可读取同一路视频流。
4. 视频流中断后系统能展示离线状态。

## AI 上报测试要求

1. 上报 JSON 字段完整。
2. `cameraId` 可匹配已存在摄像头。
3. `timestamp` 时区明确。
4. 后端能根据结果类型生成事件。
5. 告警类事件能进入告警中心。

## 最终演示验收流程

1. 启动后端、前端、AI 服务和视频流服务。
2. 登录系统。
3. 配置摄像头和视频流地址。
4. 进入实时监控页面并选择摄像头。
5. 绘制警戒区域并保存。
6. AI 服务读取视频并输出人员检测结果。
7. 员工经过时显示身份识别结果。
8. 陌生人进入画面时生成陌生人告警。
9. 人员进入警戒区域时生成区域入侵告警。
10. 告警中心展示告警并完成处置。
11. 事件日志可追溯事件、摄像头、时间、截图和处理记录。
12. 首页或统计页面展示事件和告警汇总。
