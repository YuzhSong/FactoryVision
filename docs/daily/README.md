# 工作日报

本目录用于存放团队成员每日工作日报。日报用于记录进度、问题、计划和风险，便于组长跟踪项目状态。

## 命名规则

建议文件名：

```text
YYYY-MM-DD-member-N.md
```

示例：

```text
2026-07-07-member-1.md
```

## 日报模板

```markdown
# YYYY-MM-DD 成员 N 工作日报

## 今日完成

- 

## 遇到问题

- 

## 明日计划

- 

## 风险说明

- 
```

## 填写规则

1. 今日完成写实际交付，不写泛泛描述。
2. 遇到问题写阻塞原因、影响范围和需要谁协助。
3. 明日计划写可验证的任务。
4. 风险说明写可能影响进度、质量、演示或集成的问题。
5. 涉及接口、数据库、AI 输出格式或部署变化时，同步提醒对应负责人更新文档。
