# PRD

## 项目名称

工厂实时视频分析监测系统  
Factory Real-time Video Analysis and Safety Monitoring System

## 项目背景

面向工厂安全生产场景，系统通过摄像头接入实时视频流，并结合人员检测、人脸识别、头盔检测、危险区域入侵检测、摔倒检测、异常跑动检测、离开/返回检测等能力，形成“视频采集、智能识别、事件生成、告警处置、日志追溯、统计分析”的业务闭环。

## 项目目标

系统整体目标包括：

1. 实时视频流接入与展示。
2. 人员检测与人数统计。
3. 员工人脸录入、人脸识别与陌生人告警。
4. 危险区域配置与入侵检测。
5. 头盔佩戴、摔倒、异常跑动等安全行为识别。
6. 告警中心、监控日志、事件截图与处置记录。
7. 考勤统计与离开/返回记录。
8. Jenkins CI/CD、Swagger / OpenAPI、OpenSpec 规范开发。

## 当前阶段

当前只做阶段 1：项目骨架搭建。

本阶段目标：

- 搭建 monorepo 基础目录结构。
- 创建 frontend、backend、ai-service 的可运行入口。
- 补齐 Jenkinsfile、OpenSpec、docs 文档骨架。
- 保留后续扩展接口，不实现复杂业务逻辑。

本阶段不做：

- 不实现真实模型推理。
- 不接入复杂业务 CRUD。
- 不写大量假数据。
- 不让前端直连数据库。
- 不让 AI 服务直接写数据库。

## 技术栈

- Frontend：Vue 3 + Vite + Element Plus + ECharts
- Backend：Django + Django REST Framework
- AI Service：Python，预留 OpenCV / YOLO / face_recognition 模块
- Database：MySQL，开发阶段允许 SQLite 临时运行
- CI/CD：Jenkins
- API 文档：Swagger / OpenAPI
- 规范：OpenSpec

## 模块划分

- `frontend`：登录页、首页看板、实时监控、告警中心、员工管理、摄像头管理、区域配置、考勤统计。
- `backend`：用户、员工、摄像头、区域、事件、考勤、AI 结果接收等业务边界。
- `ai-service`：视频流读取、人员检测、人脸识别、头盔检测、区域检测、摔倒检测、跑动检测、事件上报。
- `openspec`：项目级规范、模块规格、后续变更设计。
- `docs`：PRD、架构、API、数据库、部署、测试报告。

## 分阶段开发计划

### 阶段 0

文档与范围固化：统一项目范围、技术栈、目录结构、OpenSpec 和说明文档。

项目骨架搭建：完成 frontend、backend、ai-service、Jenkins、OpenSpec、docs 基础结构。

### 阶段 3

实时视频流展示：跑通摄像头配置、监控页与视频展示闭环。

### 阶段 4

### 阶段 5

人脸识别与陌生人告警：实现员工识别、陌生人标记与事件闭环。

### 阶段 6

危险区域绘制与入侵检测：实现区域配置、区域判断和告警联动。

告警中心与事件日志完善：统一管理普通事件、异常事件、告警事件。

头盔、摔倒、跑动检测：补齐安全异常行为检测能力。

### 阶段 9
