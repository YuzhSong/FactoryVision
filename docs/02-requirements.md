# 需求说明

## 优先级定义

| 优先级 | 定义 |
| --- | --- |
| P0 | 支撑项目主流程和最终演示的必做能力 |
| P1 | 提升安全检测完整性和演示效果的能力 |
| P2 | 工程化或展示增强能力 |

## 非功能需求

| 类别 | 要求 |
| --- | --- |
| 实时性 | 输入和输出均使用最新帧策略；前端优先 WebRTC，避免播放器缓冲持续增长 |
| 可用性 | 输入断流停止当前处理并暴露错误，不继续输出旧帧 |
| 可配置性 | 人脸、安全帽、摔倒、区域检测可独立开关，首次仅默认开启区域检测 |
| 可观测性 | AIService 状态接口区分推理、绘制、writer 和检测项耗时及丢帧指标 |
| 数据一致性 | AIService 不直接写数据库，事件与告警由 Backend 统一生成 |
| 安全性 | 业务页面使用 JWT；生产密钥、数据库密码和 Token 通过环境变量注入 |
| 可追溯性 | 事件保留时间、摄像头、trackId、bbox、置信度、payload 和媒体证据 |
| 可维护性 | 接口以 Swagger/OpenAPI 描述，需求基线以 OpenSpec 描述 |

## 验收边界

本版本以单路演示流的完整闭环为主要验收对象。检测效果受模型权重、GPU、拍摄角度、光照和多人数量影响；跨镜头追踪、活体防伪和完整考勤属于后续扩展，不影响项目结题。
