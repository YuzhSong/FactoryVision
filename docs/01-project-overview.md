# 项目总览

## 项目背景

工厂实时视频分析监测系统面向工厂安全生产场景。系统通过摄像头采集实时视频流，结合人脸识别、人员检测、头盔佩戴检测、危险区域入侵检测、摔倒检测、异常跑动检测、离开/返回检测等能力，形成“视频采集-智能识别-事件生成-告警处置-日志追溯-统计分析”的业务闭环。

传统人工巡检存在覆盖范围有限、响应延迟、日志追溯困难和统计口径不统一等问题。本项目通过视频流接入、AI 识别、事件管理和告警处置能力，将现场风险转化为可检索、可追踪、可统计的数据资产。

## 项目目标

1. 建立前端、后端、AI 服务、视频流服务、数据库相互解耦的项目架构。
2. 支持多摄像头实时视频展示和状态管理。
3. 支持人员检测、人脸识别、陌生人告警和警戒区域入侵告警。
4. 支持头盔检测、摔倒检测、异常跑动检测、离开/返回检测等安全行为识别能力扩展。
5. 建立告警中心、事件日志、监控日志和统计分析能力。
6. 建立 Swagger / OpenAPI、OpenSpec、Jenkins 和测试文档支撑的工程化交付流程。

## 系统定位

系统定位为工厂安全生产的实时视频智能分析平台。当前代码已经具备多服务目录结构、后端 API 框架、前端页面骨架、AI 服务部分算法能力和 CI/CD 脚本骨架。AI 侧人员检测、人脸识别、头盔、跌倒、跑动、危险区域、陌生人、基础活体和带框流回推已部分落地；后端持久化、告警处置和前端真实业务闭环仍需继续实现。

## 最终演示闭环

1. 管理员登录系统。
2. 在摄像头管理中配置摄像头和视频流地址。
3. 在警戒区域页面为指定摄像头绘制多边形危险区域。
4. 打开实时监控页面，选择摄像头查看视频流。
5. AI 服务读取视频帧，输出人员、人脸、区域、安全行为检测结果。
6. 后端接收 AI 上报结果，生成事件日志和告警记录。
7. 前端通过 REST API 查询历史数据，通过 WebSocket 接收实时检测消息。
8. 管理员在告警中心确认、处理并关闭告警。
9. 系统在日志和统计页面展示事件追溯、处理记录和趋势分析。
