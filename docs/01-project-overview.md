# 项目总览

## 项目背景

Factory Vision 面向工厂安全生产场景，通过摄像头或手机采集实时视频，结合人员、人脸、安全帽、摔倒和危险区域检测，形成“视频采集—智能识别—事件生成—告警处置—证据回放—日报汇总”的闭环。

系统解决人工巡检覆盖有限、响应滞后、证据分散和统计口径不统一的问题，将现场风险转化为可检索、可追溯、可处置的数据。

## 项目目标

1. 以前端、业务后端、AIService、视频流服务和数据库解耦的方式完成可部署系统。
2. 支持摄像头管理、实时带框视频和低延迟播放。
3. 支持人员检测、人脸识别、安全帽、摔倒和区域风险识别。
4. 支持事件入库、实时推送、告警处置、关键帧和短视频回放。
5. 支持首页统计与 AI 监控日报预览、Word 导出。
6. 通过 Swagger、OpenSpec、自动化测试和 Jenkins 支撑工程化交付。

## 系统边界

- Frontend 通过 Backend 的 REST API 和 WebSocket 获取业务数据，通过 SRS 地址播放视频。
- Backend 是认证、业务规则、数据持久化和告警策略的唯一入口。
- AIService 从 SRS 拉取原始 RTMP，回推带框 RTMP，并通过 Backend API 同步配置和上报结果。
- SRS 只承担视频接入和协议分发，不保存业务状态。
- 本地开发使用 SQLite；生产部署使用 PostgreSQL。
- AIService 因 GPU、模型权重和本地采集链路独立部署，不进入云端前后端 Compose。

## 最终演示闭环

1. 管理员登录并维护员工、人脸、摄像头和警戒区域。
2. 手机或采集端向 SRS 推送原始 RTMP 流。
3. 在实时监控页选择摄像头和检测开关，启动 AI 流处理。
4. AIService 持续输出人物框和已启用检测的标注，并将带框流回推 SRS。
5. 前端低延迟播放带框流，通过 WebSocket 接收实时事件。
6. Backend 保存事件及告警；告警中心展示关键帧、回放片段和原始证据。
7. 管理员处置告警，并在首页看板和 AI 监控日报中查看汇总结果。

## 交付约束

- 人物检测是视频分析基础能力，不提供关闭开关。
- 检测开关用于控制实际模型执行，不只是隐藏前端标注。
- 模型权重、运行日志、数据库和事件媒体属于运行期资产，不提交仓库。
- OpenSpec 主规格表示当前系统基线；新功能必须通过 change 提案，完成后及时归档。
